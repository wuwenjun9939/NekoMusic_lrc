# 🎵 Neko云音乐 公开曲库贡献指南

感谢你为 **Neko云音乐** 贡献歌词与封面！为了保证多端（Android/Tauri/Web）显示效果的一致性，请在提交 PR 时务必遵循以下结构标准。

---

### 📝 歌词文件 (.lrc) 格式规范

所有歌词文件必须使用 **UTF-8** 编码，并遵循以下嵌套结构：

1. **时间轴：** 使用 `[mm:ss.xxx]` 格式（毫秒位建议保留 2-3 位）。
2. **原文层：** 时间标签后直接紧跟原文内容。
3. **翻译层：** 翻译内容需紧随原文下方，并使用 `{" "}` 进行包裹。

#### 📥 提交模板示例：
```lrc
[00:00.389] ざこざこざこざこ くだらない存在 あわれだね
{"杂鱼杂鱼杂鱼杂鱼 无聊的存在 真可怜呢"}

[00:07.546] ざこざこざこざこ ざこのざこ攻撃 効かないよ
{"杂鱼杂鱼杂鱼杂鱼 杂鱼的杂鱼攻击 根本没用喔"}

[00:14.225] ざぁこ
{"杂~鱼~"}
```

### 📥 封面图片规范 (Cover Standards)

#### 1. 必选要求 (Crucial Requirements)
- **形状：** 必须是**正方形 (1:1)** 喵！
    * *拒绝长方形，否则客户端裁剪后会变丑喵！*
- **分辨率：** 建议不低于 **256 x 256** 像素，不超过 **1000 x 1000** 像素。
    * *太小的图片（例如 <400x400）在高分屏下会变糊喵。*
- **格式：** 优先接受 `.jpg` 或 `.png`。
    * *请确保图片未损坏，且不包含多余的 WebP 或 GIF 格式喵。*

#### 2. 内容质量 (Content Quality)
- **清晰度：** 拒绝模糊、全是噪点或包含水印（如其他平台的 Logo）的图片。
- **无黑边：** 请确保图片内容填充完整，不要有上下或左右的黑边。

#### 3. 命名规则 (Naming Convention)
为了方便系统自动匹配，封面文件的命名必须保持高度一致。

**格式要求：** `音乐ID.文件后缀`
- 例如`16.jpg`
